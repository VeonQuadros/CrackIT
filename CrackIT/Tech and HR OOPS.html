OOPS Interview Questions

Master key Object-Oriented Programming concepts for technical interviews

Key OOPS Questions

What is Object-Oriented Programming (OOPS)?

Object-Oriented Programming is a methodology to design programs using classes and objects, simplifying software development and maintenance.

  • Class: A user-defined data type defining properties and functions (e.g., Human as a class with body parts as properties and actions as functions).
  • Object: A runtime instance of a class, representing entities like a person or place, operating on data and functions.
  • Note: Classes don’t occupy memory until objects are instantiated.
  • C++ Class Syntax Example:

class student {
    public:
        int id; // data member
        int mobile;
        string name;
        int add(int x, int y) { // member function
            return x + y;
        }
};

Object Creation: student s = new student();

Note: Using new allocates memory in the heap; without it, the object is null in stack memory.

Sample Answer: Object-Oriented Programming (OOPS) is a programming paradigm that uses classes and objects to structure software. A class is a blueprint defining properties and behaviors, while an object is an instance of a class. For example, a "Car" class might have properties like color and methods like drive(). OOPS supports inheritance, encapsulation, polymorphism, and abstraction, enhancing code reusability and maintainability.

What is Inheritance?

Inheritance allows a class to inherit properties and behaviors from another class, promoting code reuse and extensibility.

  • Derived Class: Inherits from a base class.
  • Base Class: The class whose members are inherited.
  • Visibility Modes: private, protected, public.
  • Types:
    • Single: One class inherits another.
    • Multiple: A class inherits from multiple classes.
    • Hierarchical: Multiple classes inherit from one base class.
    • Multilevel: A class is derived from another derived class.
    • Hybrid: Combination of multiple and hierarchical inheritance.
  • C++ Syntax: class derived_class : visibility-mode base_class;

Sample Answer: Inheritance enables a derived class to inherit attributes and methods from a base class, promoting code reuse. For example, a "Vehicle" base class can have a "Car" derived class inheriting properties like speed, adding specific features like model type. C++ supports single, multiple, hierarchical, multilevel, and hybrid inheritance, with visibility modes (public, private, protected) controlling access to inherited members.

What is Encapsulation?

Encapsulation combines data and functions into a class, hiding data from direct access using getter/setter methods, enabling data hiding.

  • Data is kept private; accessed via public methods.
  • Reduces dependencies and supports data hiding (e.g., using protected/private in C++).

Sample Answer: Encapsulation bundles data and methods into a class, restricting direct data access to ensure security. For example, in a "BankAccount" class, the balance is private, accessed via public methods like getBalance(). This promotes data hiding, reducing dependencies and enhancing maintainability. In C++, private and protected access specifiers enforce encapsulation.

What is Abstraction?

Abstraction provides an abstract view of a problem, hiding unnecessary details and focusing on essential properties and operations.

  • Data Binding: Connects application UI with business logic, reflecting changes instantly.
  • Simplifies complex problems by modeling standard solutions.

Sample Answer: Abstraction simplifies real-world problems by focusing on essential features while hiding irrelevant details. For example, in a "Car" class, users interact with methods like drive() without needing to know engine mechanics. Abstraction supports data binding, where UI changes reflect business logic updates, and enables reusable models for problems with similar properties, improving efficiency.

What is Polymorphism?

Polymorphism allows different classes to share the same interface with varying implementations, supporting multiple forms.

  • Compile-Time (Static): Via method overloading or operator overloading.
  • Runtime (Dynamic): Via method overriding using virtual functions.
  • Method Overloading Example:

#include 
using namespace std;
class Add {
    public:
        int add(int a, int b) {
            return (a + b);
        }
        int add(int a, int b, int c) {
            return (a + b + c);
        }
};
int main() {
    Add obj;
    int res1 = obj.add(2, 3);
    int res2 = obj.add(2, 3, 4);
    cout << res1 << " " << res2 << endl;
    return 0;
}

Output: 5 9

Sample Answer: Polymorphism, meaning "many forms," allows classes to share interfaces with different implementations. Compile-time polymorphism, like method overloading, uses different parameter lists for the same function name, resolved at compile time. Runtime polymorphism, achieved via virtual functions and overriding, determines the method to call at runtime based on the object’s type, enabling flexible inheritance hierarchies.

Runtime Polymorphism Example:


#include 
using namespace std;
class Base {
public:
    virtual void show() {
        cout << "Apni Kaksha base" << endl;
    }
};
class Derived : public Base {
public:
    void show() {
        cout << "Apni Kaksha derived" << endl;
    }
};
int main() {
    Base* b;
    Derived d;
    b = &d;
    b->show(); // prints "Apni Kaksha derived"
    return 0;
}

What are Constructors and Destructors?

Constructors initialize objects, while destructors clean up resources when objects are destroyed.

  • Default Constructor: No parameters, invoked automatically.
  • Parameterized Constructor: Takes parameters for custom initialization.
  • Copy Constructor: Initializes an object using another object.
  • Destructor: Cleans up resources, prefixed with ~, invoked automatically.

#include 
using namespace std;
class go {
public:
    int x;
    go(int a) { // parameterized constructor
        x = a;
    }
    go(go &i) { // copy constructor
        x = i.x;
    }
};
int main() {
    go a1(20);
    go a2(a1);
    cout << a2.x << endl;
    return 0;
}

Output: 20

Destructor Example:


#include 
using namespace std;
class A {
public:
    A() {
        cout << "Constructor in use" << endl;
    }
    ~A() {
        cout << "Destructor in use" << endl;
    }
};
int main() {
    A a;
    return 0;
}

Sample Answer: Constructors are special methods invoked automatically to initialize objects. Default constructors take no parameters, parameterized constructors accept arguments, and copy constructors initialize from another object. Destructors, prefixed with ~, are called automatically to free resources when an object is destroyed, ensuring proper cleanup, like deallocating memory.

What is a Friend Function?

A friend function is a non-member function that can access a class’s private and protected members.

  • Declared in the class but defined outside.
  • Uses object name and dot operator to access members.
  • Takes objects as arguments.

Sample Answer: A friend function is a non-member function granted access to a class’s private and protected members. It’s declared inside the class with the friend keyword but defined outside, accessing members via objects and the dot operator. For example, a friend function can access a private variable like balance in a "BankAccount" class, useful when external functions need controlled access to class data.

What is a Virtual Function?

A virtual function enables runtime polymorphism by allowing derived classes to override base class implementations.

  • Declared with the virtual keyword in the base class.
  • Resolved at runtime based on the object type.
  • Cannot be static; destructors can be virtual, but not constructors.

#include 
using namespace std;
class base {
public:
    virtual void print() {
        cout << "print base class" << endl;
    }
    void show() {
        cout << "show base class" << endl;
    }
};
class derived : public base {
public:
    void print() {
        cout << "print derived class" << endl;
    }
    void show() {
        cout << "show derived class" << endl;
    }
};
int main() {
    base* bptr;
    derived d;
    bptr = &d;
    bptr->print(); // prints "print derived class"
    bptr->show(); // prints "show base class"
    return 0;
}

Sample Answer: A virtual function, marked with virtual in a base class, allows derived classes to override it, enabling runtime polymorphism. The function called depends on the object’s type, not the pointer type. For example, a base class pointer to a derived object calls the derived class’s virtual function, ensuring dynamic behavior. Virtual functions cannot be static, and destructors can be virtual, but not constructors.

What is a Pure Virtual Function?

A pure virtual function is a placeholder in a base class with no implementation, making the class abstract.

  • Declared with = 0.
  • Abstract classes cannot be instantiated.
  • Derived classes must implement the function.

#include 
using namespace std;
class Shape {
public:
    virtual void draw() = 0;
};
class Rectangle : public Shape {
public:
    void draw() {
        cout << "Rectangle" << endl;
    }
};
class Square : public Shape {
public:
    void draw() {
        cout << "Square" << endl;
    }
};
int main() {
    Shape* s1 = new Rectangle();
    Shape* s2 = new Square();
    s1->draw();
    s2->draw();
    return 0;
}

Output: Rectangle
Square

Sample Answer: A pure virtual function, declared with = 0 in a base class, has no implementation and makes the class abstract, preventing instantiation. Derived classes must provide implementations. For example, a "Shape" class with a pure virtual draw() ensures derived classes like "Rectangle" define their own drawing logic, supporting runtime polymorphism.

What are Namespaces?

Namespaces provide a logical division of code to prevent naming conflicts.

  • Define scope for identifiers like variables, classes, and functions.
  • Prevent ambiguity when multiple entities share the same name.
  • C++’s standard namespace is std, included via using namespace std;.

#include 
using namespace std;
namespace Add {
    int a = 5, b = 5;
    int add() {
        return (a + b);
    }
}
int main() {
    int res = Add::add();
    cout << res;
    return 0;
}

Output: 10

Sample Answer: Namespaces logically group code to avoid naming conflicts. For example, two functions named add() can exist in different namespaces without ambiguity. C++’s std namespace includes standard library components. Using namespace Add, you can define variables and functions, accessed via the scope resolution operator ::, ensuring clarity and organization.

What are Access Specifiers?

Access specifiers define how class members can be accessed from outside the class.

  • Private: Accessible only within the class.
  • Public: Accessible from anywhere.
  • Protected: Accessible within the class and its derived classes.

Sample Answer: Access specifiers control access to class members. Private members are accessible only within the class, public members are accessible everywhere, and protected members are accessible in the class and its derived classes, commonly used in inheritance. For example, a private variable in a "Car" class can only be accessed via public methods, ensuring controlled access and encapsulation.

What is the Difference Between Overloading and Overriding?

Overloading and overriding are mechanisms to achieve polymorphism but differ in scope and binding.

  • Overloading: Static binding; same method name with different parameters in the same class.
  • Overriding: Dynamic binding; same method name and signature in base and derived classes.
  • Key Notes:
    • delete releases a single object; delete[] releases an array.
    • Virtual inheritance ensures one copy of a base class in multiple inheritance.

Sample Answer: Overloading occurs within a class, where multiple methods share the same name but differ in parameters, resolved at compile time (static binding). For example, multiple add() methods with different argument counts. Overriding occurs when a derived class redefines a base class’s virtual method with the same signature, resolved at runtime (dynamic binding). This enables runtime polymorphism, like a derived class overriding a base class’s virtual function.
